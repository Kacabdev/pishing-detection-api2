# Phishing Website Detection API

https://kacabdev.github.io/pishing-detection-api2/
A machine learning system that classifies websites as **phishing** or **legitimate** from 30 structural features of their URL and page — no live crawling required. Built as a final capstone project: dataset → preprocessing → model comparison → deployed API → UI.

## Overview

Phishing sites imitate trusted domains to steal credentials. Attackers change the content, but structural signals — URL length, IP-address usage, subdomain count, SSL state, domain age, and similar — tend to stay predictable. This project trains and compares three classifiers on those signals and serves the best one behind a `/predict` endpoint, with a simple web UI on top.

## Dataset

| | |
|---|---|
| **Source** | [UCI Machine Learning Repository — Phishing Websites (ID 327)](https://archive.ics.uci.edu/dataset/327/phishing+websites) |
| **Raw size** | 11,055 rows × 30 features |
| **After cleaning** | 5,849 rows (5,206 exact duplicates removed) |
| **Target** | `result` — recoded to `1 = phishing`, `0 = legitimate` |
| **Features** | All pre-encoded as `{-1, 0, 1}` — address bar, abnormal-based, HTML/JS, and domain-based indicators |

## Algorithms Compared

All three models trained on the identical stratified 80/20 split, evaluated on the same held-out test set.

| Algorithm | Accuracy | Precision | Recall | F1-Score |
|---|---|---|---|---|
| **XGBoost** (winner) | 0.9479 | 0.9594 | 0.9387 | **0.9490** |
| Random Forest | 0.9402 | 0.9572 | 0.9255 | 0.9411 |
| Logistic Regression | 0.9179 | 0.9247 | 0.9156 | 0.9201 |

**Selection rule:** highest F1-Score, tie-broken by Recall. In phishing detection, a missed phishing site (low recall) puts a user at risk, while a false alarm (low precision) erodes trust — F1 balances both instead of favoring one. **XGBoost** wins on both counts and is the model deployed at `/predict`.

## API

**Endpoint:** `POST /predict`

```bash

  -d '{

    "having_sub_domain": -1, "sslfinal_state": -1, "domain_registration_length": -1,
    "favicon": 1, "port": 1, "https_token": -1, "request_url": 1,
    "url_of_anchor": -1, "links_in_tags": 1, "sfh": -1, "submitting_to_email": -1,
    "abnormal_url": -1, "redirect": 0, "on_mouseover": 1, "rightclick": 1,
    "popupwindow": 1, "iframe": 1, "age_of_domain": -1, "dnsrecord": -1,
    "web_traffic": -1, "page_rank": -1, "google_index": 1,
    "links_pointing_to_page": 1, "statistical_report": -1

```

```json
{
  "prediction": "phishing",
  "phishing_probability": 1.0,
  "legitimate_probability": 0.0,
  "model": "XGBoost"

```

`GET /health` reports the deployed model and its test-set metrics. Interactive docs are available at `/docs`.

## Frontend

A static UI at `/` (served by the same FastAPI app) groups the 30 features into readable sections, includes one-click phishing/legitimate example presets, and displays the verdict with a probability bar. No separate server or build step needed.

## Results Summary

XGBoost outperformed both bootcamp baselines, reaching 94.8% accuracy and a 0.949 F1-score on held-out data, with recall (93.9%) high enough to catch the large majority of phishing attempts while keeping false alarms low (95.9% precision). The gap over Logistic Regression suggests the feature set has non-linear interactions (e.g. between SSL state and domain age) that tree-based ensembles capture better than a linear boundary.
